# チーム開発実習用リポジトリ

このリポジトリは、**GitHub Pull Request (PR)** 実習用です。  
チームメンバー全員で `teamdata.txt` を修正し、PRを通してリーダーのリポジトリに統合します。



---

## 📄 ファイル構成

- **`README.md`** : この説明ファイル

- **`teamdata.txt`** : チームメンバーの情報を記載するファイル

  

---

## 📝 `teamdata.txt` の内容

初期状態（例）：

```
【チーム情報】
＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝
<チーム名> 自分の名前（ローマ字）
コメント：自分の趣味・特技・好きなもの（１行で）
＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝
```

---


## ✏ 修正方法

1. `<チーム名>` を自分のチーム名に置き換える  
2. `自分の名前（ローマ字）` を自分の名前に置き換える  
3. `コメント` の部分に趣味・特技・好きなものを **1行で** 記入する  
4. 必要に応じて改行やインデントは崩さないように注意

例：

```
【チーム情報】
＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝
TeamA TaroYamada
コメント：サッカー観戦とプログラミング
＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝
```

---

## 🚀 提出方法（概要）

1. **ブランチ作成**

   ```bash
   git branch br01
   git switch br01
   ```

2. **修正・保存**

3. **コミット**

```
git add teamdata.txt
git commit -m "名前とコメントを追加"
```

4. **GitHubにpush**

```
git push origin br01
```

5. **Pull Request作成**

   - GitHub上で「Compare & pull request」ボタンをクリック
   - PRタイトルと説明を記入し、作成

   

## **💡 注意**



- **他の人の記述は消さない**
- フォーマットを崩さない（全角・半角、記号の位置に注意）
- マージ後はリーダの指示で同期を行う
